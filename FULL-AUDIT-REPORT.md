### CRITICAL: Domain Points to Wrong Site

**goodhairbyzane.com is currently serving a WordPress site, NOT the Next.js app in this repository.**

Evidence:
- robots.txt references `/wp-admin/`
- Sitemap generated by "All in One SEO v4.9.5.1"
- WordPress 6.9.4 detected with "Extendable" theme
- Page structure differs completely from our codebase (separate /sluzby/, /kontakt/, /galeria/, /o-nas/ pages)
- The live H1 is "Luxusná starostlivosť pre zdravé a krásne vlasy" — different from our Next.js hero

**Impact:** Our Next.js app with all its SEO optimizations is NOT being served to users or search engines.

### Top 5 Critical Issues

1. **Domain not pointing to Next.js app** — All SEO work in this codebase is invisible to Google
2. **No OG image exists** — `/og-image.jpg` referenced in metadata but file doesn't exist
3. **Missing security headers on live site** — No HSTS, CSP, or X-Content-Type-Options (WordPress)
4. **Large images still in original format** — 5 HDR JPGs couldn't be converted (already optimized)
5. **No `lang` attribute consistency** — Live WordPress site uses `sk-SK`, our app uses `sk`

### Top 5 Quick Wins

1. Point domain to Vercel deployment (immediate SEO impact)
2. Create OG image from salon photos (social sharing)
3. Add HSTS header to next.config.js
4. Add `next/image` width/height to prevent CLS
5. Add `rel="noopener"` to all external links (already done — good)
